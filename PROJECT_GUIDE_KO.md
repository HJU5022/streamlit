# ModelGuard AI 초보자 설명서

## 1. app.py와 src 폴더는 어떤 관계인가요?

`app.py`는 공연의 무대입니다. `src` 안의 파일들은 데이터, 모델, 그래프, Agent를 담당하는 스태프입니다. 실행은 `app.py` 하나만 하지만 내부에서는 여러 파일의 함수를 불러옵니다.

```text
app.py
 ├─ data.py: CSV를 읽고 나눔
 ├─ modeling.py: 모델을 학습하고 예측
 ├─ diagnostics.py: 실제 숫자를 계산
 ├─ scoring.py: 건강점수 계산
 ├─ charts.py: 그래프 생성
 └─ agent.py: Tool 결과를 조치사항으로 정리
```

파일을 나눈 이유는 오류 위치를 찾기 쉽고, 기능을 재사용하고, 테스트하기 쉽기 때문입니다.

## 2. Streamlit의 rerun은 무엇인가요?

버튼이나 Selectbox를 바꾸면 Streamlit은 `app.py`를 위에서 아래로 다시 실행합니다. 모델을 매번 학습하면 느리므로 모델은 `cache_resource`, 데이터와 계산 결과는 `cache_data`, 화면에서 기억할 값은 `session_state`를 사용합니다.

## 3. 데이터 드리프트란 무엇인가요?

모델이 학습한 과거 데이터와 현재 데이터의 분포가 달라지는 현상입니다. 예를 들어 과거에는 상품 L이 30%였지만 현재 70%가 되었다면 모델이 익숙하지 않은 환경일 수 있습니다.

- 숫자형: PSI와 KS 통계량
- 범주형: 카테고리 비율과 JS divergence

## 4. Recall과 F1은 무엇인가요?

- Recall: 실제 양성 중 모델이 찾아낸 비율입니다. 놓치면 큰 문제가 되는 업무에서 중요합니다.
- Precision: 모델이 양성이라고 한 것 중 실제 양성 비율입니다.
- F1: Precision과 Recall을 함께 고려합니다.

Accuracy만 보면 불균형 데이터에서 모델이 좋아 보일 수 있어 여러 지표를 함께 표시합니다.

이 설명은 이진 분류 모델에 해당합니다. 다중 분류에서는 클래스별 결과를 같은 비중으로 보는 Macro F1을 사용하고, 회귀 분석에서는 실제값과 예측값의 차이인 MAE·RMSE와 설명력인 R²를 사용합니다.

## 4-1. 앱이 모델 유형을 어떻게 고르나요?

- 정답이 두 종류면 `이진 분류 (Binary Classification)`
- 정답이 3~100종류면 `다중 분류 (Multiclass Classification)`
- 숫자형 정답의 고유값이 충분히 많으면 `회귀 분석 (Regression)`

원클릭 데모에서는 자동으로 설정합니다. CSV 업로드에서는 자동으로 선택된 모델 유형을 보여주며, 필요한 경우 사용자가 직접 변경할 수 있습니다.

## 5. 혼동행렬은 어떻게 읽나요?

실제 0/1과 예측 0/1을 네 칸으로 나눈 표입니다. 오탐(False Positive)과 미탐(False Negative)이 어디서 늘었는지 기준과 현재 행렬을 비교합니다.

## 6. Calibration은 무엇인가요?

모델이 80% 확률이라고 말한 사례가 실제로도 약 80% 맞는지를 확인합니다. Brier Score는 작을수록 좋습니다. 확률이 지나치게 자신만만하면 의사결정이 위험해질 수 있습니다.

## 7. 취약 구간은 무엇인가요?

전체 성능은 괜찮아도 특정 상품·지역 그룹에서 Recall이 낮거나 오탐률이 높을 수 있습니다. 이 앱은 범주형 그룹별 성능을 비교하지만 표본이 30건 미만이면 확정하지 않습니다.

## 8. 건강점수는 어떻게 계산하나요?

데이터 품질, 드리프트, 성능, 확률 신뢰성, 취약 구간 점수를 가중 평균합니다. 분석할 수 없는 항목은 0점으로 벌주지 않고 남은 항목끼리 다시 가중합니다. 이 점수는 인증이 아닌 우선순위 지표입니다.

## 9. 왜 AI Agent인가요?

숫자는 Python Tool이 계산합니다. LLM Agent는 필요한 Tool을 선택하고 결과를 관찰한 뒤 조치를 정리합니다. 키가 없을 때는 규칙 기반 보고서이며, 앱도 이를 정직하게 표시합니다.

## 10. 데이터 누수는 어떻게 막았나요?

전체 데이터를 전처리한 뒤 나누지 않습니다. 기준 데이터의 학습 구간에만 전처리기와 모델을 fit하는 scikit-learn Pipeline을 사용합니다. ID와 누수 후보도 화면에 표시합니다.

## 11. 면접 예상 질문

### Q. 왜 Accuracy 하나만 사용하지 않았나요?

클래스 불균형에서는 모두 0이라고 예측해도 Accuracy가 높을 수 있어 Recall, Precision, F1, ROC-AUC, PR-AUC, 오탐·미탐을 함께 비교했습니다.

### Q. LLM이 잘못된 숫자를 만들면 어떻게 하나요?

LLM은 숫자를 계산하지 않습니다. 허용된 Python Tool 결과만 받아 설명하도록 시스템 프롬프트와 도구 레지스트리를 분리했습니다.

### Q. 건강점수는 과학적 기준인가요?

아닙니다. 점검 우선순위를 정하기 위한 내부 평가 지표이며 구성 점수와 감점 근거를 사용자에게 보여줍니다.

### Q. 실제 서비스에서는 무엇이 더 필요한가요?

실시간 로그 수집, 데이터 저장소, 모델 버전 관리, 라벨 지연 처리, 알림, 비용 기반 임곗값, 인증, 모니터링 이력과 승인 절차가 필요합니다.

## 12. 내가 직접 기억할 핵심 문장
